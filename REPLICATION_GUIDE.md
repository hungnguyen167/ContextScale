# Replication Guide — ContextScale

This guide provides step-by-step instructions to fully replicate all analyses in:

> Nguyen, Hung H. V. "A Sentiment-Based Approach to Measuring Multidimensional Party Positions using Transformer." *Political Science Research and Methods* (2026).

The complete codebase, data, and trained model weights are archived on the **[PSRM Harvard Dataverse](https://dataverse.harvard.edu/dataverse/PSRM)**.

## 1. Repository Layout

```
ContextScale/
├── 01_manifesto_pull.Rmd       # Step 1 (R): Pull manifesto data from CMP API
├── 02_r_prep.Rmd               # Step 2 (R): CHES cleaning + coalition agreement prep  [before Step 3]
├── 03_main_analyses.ipynb      # Step 3 (Python): Training, evaluation, scaling
├── 04_r_prep.Rmd               # Step 4 (R): Wordfish benchmarks                        [after  Step 3]
├── 05_visualizations.Rmd       # Step 5 (R): Figures and tables
├── train.py                    # Standalone CLI training script
├── requirements.txt            # Python dependencies (pinned versions)
├── environment.yml             # Conda environment specification
├── utils/
│   ├── functions.py            # Preprocessing, training loops, evaluation
│   ├── models.py               # ContextScalePrediction architecture
│   └── uncertainty.py          # Deep ensemble & uncertainty estimation
├── data/
│   ├── ches/                   # CHES expert survey data (v3, 1999–2019)
│   ├── CMP/                    # CMP party/manifesto metadata tables
│   ├── coalitionagree/         # Hand-coded coalition agreement texts
│   ├── MOTN/                   # Twitter ground-truth datasets
│   ├── r_outputs/              # Generated by Steps 1–2 (NOT in repo, see §3)
│   └── py_outputs/             # Generated by Step 3 (NOT in repo, see §3)
└── results/
    ├── classification results/ # Per-variant F1/precision/recall tables
    ├── datasets/               # Final party position score datasets
    ├── models/                 # Trained model weights (.safetensors)
    └── tabs and figs/          # Paper figures and tables
```

> **Note:** `data/r_outputs/`, `data/py_outputs/`, `data/temps/`, and `results/` are excluded from the Git repository. They are either generated by running the pipeline or available for download from Dataverse (see §7).

The repository includes an [`renv.lock`](renv.lock) file with exact versions of all R packages. To restore the environment:

```r
# Install renv if needed
install.packages("renv")

# Restore all R packages to the pinned versions in renv.lock
renv::restore()
```

`renv` will install every package at the exact version recorded in the lockfile into an isolated project library. This is the recommended approach for full reproducibility.

If you prefer a manual install without `renv`:

```r
install.packages("pacman")
pacman::p_load(tidyverse, manifestoR, quanteda, quanteda.textmodels,
               ggplot2, viridis, ggsci, ggpubr, ragg, scales, gt,
               rstatix, gridExtra, openxlsx, countrycode, scico, caret,
```

## 4. Setting Up the Environment

### Option A — Conda (recommended)
```bash
conda env create -f environment.yml
conda activate contextscale
```

For GPU training with CUDA 13.0, replace the torch entry after creating the environment:
```bash
pip install torch==2.9.0+cu130 torchvision==0.24.0+cu130 \
    --index-url https://download.pytorch.org/whl/cu130
```

### Option B — pip with virtualenv
```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Post-install: spaCy & NLTK language data
```bash
python -m spacy download en_core_web_sm
python -c "import nltk; nltk.download('stopwords')"
```

## 5. Step-by-Step Replication

Run steps in this order: **1 → 2 → 3 → 4 → 5**.

Steps 1, 2, 4, and 5 use R; Step 3 uses Python. `04_r_prep.Rmd` (Wordfish) runs after Step 3 because its models depend on preprocessed data the Python notebook produces.

### Step 1 — Pull Manifesto Data (`01_manifesto_pull.Rmd`)

**Prerequisites:** CMP API key in `manifesto_apikey.txt` (see §3b), R packages installed.

Open in RStudio and knit, or run via command line:
```bash
Rscript -e "rmarkdown::render('01_manifesto_pull.Rmd')"
```

**Outputs:**
- `data/r_outputs/pulled_manifestoes.csv` — 17-country training corpus
- `data/r_outputs/pulled_manifestoes_test.csv` — 5-country out-of-sample test corpus

> Skip this step and use the pre-generated CSVs from Dataverse if you do not wish to register for an API key.

---

### Step 2 — R Preparation (`02_r_prep.Rmd`)

**Prerequisites:** Output from Step 1 (`pulled_manifestoes.csv`).
```bash
Rscript -e "rmarkdown::render('02_r_prep.Rmd')"
```

**Outputs:**
- `data/r_outputs/ches_cleaned.csv`
- `data/r_outputs/coalitionagree_texts.csv` ← required by `03_main_analyses.ipynb` §3.7

---

### Step 3 — Main Analyses (`03_main_analyses.ipynb`)

**Prerequisites:** Conda environment activated, GPU available, outputs from Steps 1 and 2a.

```bash
conda activate contextscale
jupyter lab 03_main_analyses.ipynb
```

Run cells **sequentially**. Training cells (3.2, 3.3, 3.5) are the most time-intensive.

**Alternatively, use `train.py` for long-running training in a `tmux` session:**
```bash
tmux new-session -d -s training
tmux attach-session -t training
conda activate contextscale
python train.py --num_models 5 --epochs 5
# Detach: Ctrl+B, D
```

Expected runtimes (RTX 3090 / A100):
| Section | Runtime |
|---------|---------|
| 3.1 Data preparation | ~5 min |
| 3.2 Ensemble training (5 × 5 epochs) | ~2–4 h |
| 3.3 Ablation variants (4 models) | ~4–6 h |
| 3.4 Validity checks | ~30 min |
| 3.5 Twitter fine-tuning | ~1 h |
| 3.6 Doc2Vec | ~20 min |
| 3.7 Full dataset scaling | ~3 h |

> To skip training entirely, download pre-trained weights from Dataverse (see §6) and run only the inference/scaling cells.

## 6. Pre-Trained Model Weights

Trained model weights (`.safetensors`) are **not stored in the Git repository** due to file size, but are available on [Harvard Dataverse](https://dataverse.harvard.edu/dataverse/PSRM).

### Available model packages on Dataverse

| Package | Path in repo | Description |
|---------|-------------|-------------|
| `ensemble_weights.zip` | `results/models/ensemble/` | Main 5-member ensemble (Section 3.2) |
| `ablation_weights.zip` | `results/models/manifesto_ContextScalePrediction_*/` | base / sf / sa / dg variants (Section 3.3) |
| `twitter_ensemble_weights.zip` | `results/models/tw_ensemble/` | Twitter-adapted ensemble (Section 3.5) |
| `full_released_weights.zip` | `results/models/ensemble_scaling/` | Final model trained on full dataset (Section 3.7) |
| `coalitionagree_ensemble_weights.zip` | `results/models/coalitionagree_ensemble_10/` | Coalition agreement ensemble (Section 3.7) |

### Downloading weights
```bash
# Example: using the Dataverse CLI tool (dataverse-client-python)
pip install dataverse-client-python
dvupload --doi "doi:10.7910/DVN/XXXXX" --file ensemble_weights.zip
# Then unzip into results/models/
```

Or download manually from the Dataverse web interface and unzip into `results/models/`.

### Loading weights in Python
```python
from safetensors.torch import load_file
import torch

state_dict = load_file("results/models/ensemble/model_ensemble_0.safetensors")
model.load_state_dict(state_dict)
```

## 8. Known Issues

| Issue | Fix |
|-------|-----|
| `torch` CUDA version mismatch | Re-install `torch` with the wheel matching your CUDA version (see §4) |
| `spacy` `STOP_WORDS` import error | `pip install spacy && python -m spacy download en_core_web_sm` |
| NLTK `stopwords` not found | `python -c "import nltk; nltk.download('stopwords')"` |
| `manifestoR` API timeout | CMP API can be slow; retry or download CSVs from Dataverse |
| Out-of-memory during training | Reduce `--batch_size` (default 16) or switch to `xlm-roberta-base` |
