# Replication Guide — ContextScale

This guide provides step-by-step instructions to fully replicate all analyses in:

> Nguyen, Hung H. V. "A Sentiment-Based Approach to Measuring Multidimensional Party Positions using Transformer." *Political Science Research and Methods* (2026).

The complete codebase, data, and trained model weights are archived on the **[PSRM Harvard Dataverse](https://dataverse.harvard.edu/dataverse/PSRM)**.

The repository includes an [`renv.lock`](renv.lock) file with exact versions of all R packages. To restore the environment:

```r
# Install renv if needed
install.packages("renv")

# Restore all R packages to the pinned versions in renv.lock
renv::restore()
```

`renv` will install every package at the exact version recorded in the lockfile into an isolated project library. This is the recommended approach for full reproducibility.

If you prefer a manual install without `renv`:

```r
install.packages("pacman")
pacman::p_load(tidyverse, manifestoR, quanteda, quanteda.textmodels,
               ggplot2, viridis, ggsci, ggpubr, ragg, scales, gt,
               rstatix, gridExtra, openxlsx, countrycode, scico, caret,
               ggExtra, rempsyc, flextable, magrittr, grid, readr, dplyr)
```

### Hardware
A CUDA-capable GPU (≥ 16 VRAM) is strongly recommended for Steps 3a and 3b. Replicators may change the batch size if there is Out-Of-Memory error. Changing batch size is not expected to change inference results, but may very slightly change training outcomes (negligible).

## 5. Step-by-Step Replication

Run steps in this order: **1 → 2 → 3a → 3b → 4 → 5**.

Steps 1, 2, 4, and 5 use R; Steps 3a and 3b use Python. `04_r_prep.Rmd` (Wordfish) runs after Step 3a because its models depend on preprocessed data the Python notebook produces.

### Step 1 — Pull Manifesto Data (`01_manifesto_pull.Rmd`)

**Prerequisites:** CMP API key in `manifesto_apikey.txt` (see §3b), R packages installed.

Open in RStudio and knit, or run via command line:
```bash
Rscript -e "rmarkdown::render('01_manifesto_pull.Rmd')"
```

**Outputs:**
- `data/r_outputs/pulled_manifestoes.csv` — 17-country training corpus
- `data/r_outputs/pulled_manifestoes_test.csv` — 5-country out-of-sample test corpus

> Skip this step and use the pre-generated CSVs from Dataverse if you do not wish to register for an API key.

- `data/r_outputs/coalitionagree_texts.csv` ← required by `03a_main_analyses.ipynb` §3.6

### Step 3a — Main Analyses (`03a_main_analyses.ipynb`)

**Prerequisites:** 
- Conda environment activated
- Outputs from Steps 1 and 2 (`data/r_outputs/pulled_manifestoes.csv`, etc.)
- **Pre-trained model weights** in `results/models/ensemble/` and `results/models/ensemble_scaling/` (see §7 to download, or train from scratch below)

#### Option A — Use pre-trained weights (recommended)

Download weights from Dataverse (see §7), then open the notebook:

```bash
conda activate contextscale
jupyter lab 03a_main_analyses.ipynb
```

#### Option B — Train from scratch with `train.py`

**Train the two manifesto ensembles before running the notebook.** Both require a GPU and ~2–4 hours each.

```bash
tmux new-session -d -s cs-train
tmux attach-session -t cs-train
conda activate contextscale

# 1. Evaluation ensemble (train/eval split) → results/models/ensemble/
python train.py

# 2. Full-dataset scaling ensemble → results/models/ensemble_scaling/
#    (run after step 1 completes; also required before the notebook's 10%-data section)
python train.py --train_for_scaling --save_dir results/models/ensemble_scaling

# Detach: Ctrl+B then D | Re-attach: tmux attach-session -t cs-train
```

The three additional ensembles below are trained by cells **within the notebook itself** (no separate `train.py` call needed):

| Notebook section | Model folder | Training data | Prerequisite |
|---|---|---|---|
| 3.2 (10% labeled data) | `manifesto_ensemble_dl_10/` | 10% of manifesto corpus | `ensemble_scaling/` weights |
| 3.4 (Twitter) | `tw_ensemble/` | Twitter ground-truth datasets | `ensemble_scaling/` weights |
| 3.6 (Coalition agreements) | `coalitionagree_ensemble_10/` | Coalition agreement texts | — |

Run cells **sequentially**. Approximate runtimes (GPU):

| Section | Description | Runtime |
|---------|------------|----------|
| 3.1 | Data preparation & preprocessing | ~5 min |
| 3.2 | Ensemble inference + 10%-data fine-tuning | ~2–4 h |
| 3.3 | Validity checks (out-of-sample languages) | ~30 min |
| 3.4 | Twitter fine-tuning + inference | ~30 min |
| 3.5 | Doc2Vec (Rheault & Cochrane 2020 baseline) | ~30 min |
| 3.6 | Coalition agreement fine-tuning + full scaling | ~1 h |

### Step 3b — Ablation Study (`03b_ablation_study.ipynb`)

**Prerequisites:**
- Conda environment activated
- Outputs from Step 1 (`data/r_outputs/pulled_manifestoes.csv`)
- Pre-trained weights in `results/models/ensemble_scaling/` (used to initialise ablation variants)

This notebook trains four architecture variants from the pre-trained weights and evaluates each on the manifesto corpus. Training cells run inside the notebook.

| Variant | Architecture | Save path |
|---|---|---|
| `base` | XLM-RoBERTa + two independent heads | `results/models/manifesto_ContextScalePrediction_base/` |
| `sf` | Simple unidirectional information flow | `results/models/manifesto_ContextScalePrediction_sf/` |
| `sa` | **Shared attention** (paper’s proposed model) | `results/models/manifesto_ContextScalePrediction_sa/` |
| `dg` | Dynamic gating between heads | `results/models/manifesto_ContextScalePrediction_dg/` |

```bash
conda activate contextscale
jupyter lab 03b_ablation_study.ipynb
```

Expected runtime: **~4–6 hours** total (four variants × ~1–2 h each on GPU).

## 6. Convenience Scripts

Two bash scripts in `scripts/` automate the most common tasks.
They must be run from the repository root.

### 6a. `scripts/clean.sh` — Reset to a fresh state

Deletes every pipeline-generated output while keeping raw data (`data/CMP/`,
`data/coalitionagree/`, `data/MOTN/`) and pre-trained model
weights (`results/models/`) intact.

```bash
# Dry-run (lists files that would be deleted — safe to run anytime):
bash scripts/clean.sh

# Actually delete generated outputs:
bash scripts/clean.sh --confirm
```

What gets removed:

| Path | Removed by |
|------|-----------|
| `data/r_outputs/*` | Generated by Steps 1, 2, 4 |
| `data/py_outputs/*` | Generated by Step 3a |
| `results/classification results/*` | Generated by Steps 3a/3b |
| `results/datasets/*` | Generated by Step 3a |
| `results/tabs and figs/*` | Generated by Step 5 |

### 6b. `scripts/replicate.sh` — Run the full pipeline

Handles environment creation/activation, R package installation, and runs all
pipeline steps in order.

```bash
# Full run — creates Conda env + installs R packages + runs all steps:
bash scripts/replicate.sh

# Use pip + virtualenv instead of Conda:
bash scripts/replicate.sh --venv

# Environment already created and activated — skip env setup:
bash scripts/replicate.sh --skip-env

# R packages already installed — skip renv::restore():
bash scripts/replicate.sh --skip-r-install

# Skip Step 3b ablation notebook:
bash scripts/replicate.sh --skip-ablation

# Run only Step 5 (figures & tables) — requires all prior steps completed:
bash scripts/replicate.sh --figures-only

# Run in a detachable tmux session (recommended for long runs):
bash scripts/replicate.sh --tmux

# Flags can be combined freely:
bash scripts/replicate.sh --tmux --skip-env --skip-r-install --skip-ablation
```

A timestamped log of all output is written to `logs/replicate_YYYYMMDD_HHMMSS.log`.

### Running in tmux (recommended for remote/long sessions)

Steps 3a/3b take several hours on GPU. Using `--tmux` lets you start the pipeline,
detach from the terminal, and return later — ideal for SSH sessions or overnight
runs.

```bash
# Start the pipeline in the background tmux session:
bash scripts/replicate.sh --tmux --skip-ablation

# The script prints these commands and exits immediately:
#   Session:  contextscale-rep
#   Attach:   tmux attach-session -t contextscale-rep
#   Detach:   Ctrl-B  then  D   (while attached)
#   Kill:     tmux kill-session -t contextscale-rep
#   Follow log (once pipeline starts):
#     tail -f logs/replicate_*.log

# Attach to watch progress:
tmux attach-session -t contextscale-rep

# Detach and leave running:
# Press  Ctrl-B  then  D

# Follow the log from any terminal (no attach needed):
tail -f logs/replicate_*.log

# Check if the session is still running:
tmux ls

# After the pipeline finishes the session stays open;
# kill it when you're done:
tmux kill-session -t contextscale-rep
```

> **Manual tmux** (without the `--tmux` flag):
> ```bash
> tmux new-session -s contextscale-rep
> conda activate contextscale          # or source .venv/bin/activate
> bash scripts/replicate.sh --skip-ablation
> # Ctrl-B  D  to detach
> ```

### Typical replicator workflow

```bash
# 1. Obtain pre-trained model weights from Dataverse (see §7) and unzip:

```

## 7. Pre-Trained Model Weights

Trained model weights (`.safetensors`) are **not stored in the Git repository** due to file size, but are available on [PSRM Dataverse](https://doi.org/10.7910/DVN/AFBN8X).


### Downloading weights
```bash
# Example: using the Dataverse CLI tool (dataverse-client-python)
pip install dataverse-client-python
dvupload --doi "doi:10.7910/DVN/XXXXX" --file ensemble_weights.zip
# Then unzip into results/models/
```

Or download manually from the Dataverse web interface and unzip into `results/models/`.

### Loading weights in Python
```python
from safetensors.torch import load_file
import torch

state_dict = load_file("results/models/ensemble/model_ensemble_0.safetensors")
model.load_state_dict(state_dict)
```
